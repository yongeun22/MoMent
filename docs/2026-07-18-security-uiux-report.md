# MoMent 보안 조치 및 UI/UX 개선 보고서

작성일: 2026-07-18

## 범위

이 보고서는 로컬 관리자 앱, 정적 공개 전시, Cloudflare Pages Functions를 대상으로 한 보안 점검 결과와 UI/UX 개선 우선순위를 정리한다. 사용자 요청에 따라 다음 개인정보 안내 항목은 이번 범위에서 제외했다.

- 이름·소속 공개에 대한 개인정보 안내
- 개인정보 보유 기간
- 삭제 요청 경로

## 보안 조치 결과

### 코드와 로컬 운영에 반영 완료

- 로컬 관리자 서버는 기본적으로 `127.0.0.1`에만 바인딩하고, 비루프백 주소는 명시적 예외 설정 없이는 거부한다.
- 관리자 변경 요청에 동일 출처 `Origin` 또는 `Referer` 검사를 적용했다.
- 관리자 세션 쿠키에 `HttpOnly`, `SameSite=Strict`, HTTPS 조건부 `Secure` 정책을 적용했다.
- 로그인 실패 제한을 IP+사용자명 및 IP 전체 단위로 적용했다.
- 서버 식별 정보와 오류 로그의 민감한 쿼리 문자열 노출을 줄였다.
- 정적 배포에 HSTS, 제한적 CSP, 실제 `404.html`, 자산별 캐시 정책을 포함했다.
- 공개 JSON에서 원본 파일명과 내부 생성·수정 시각을 제외했다.
- 운영자 토큰의 커밋된 기본값을 제거하고 고엔트로피 토큰의 해시 검증 방식으로 전환했다.
- Cloudflare D1 쓰기 API에 원자적 속도 제한을 추가했다.
- 방문 수는 장기 익명 브라우저 토큰으로 중복을 완화하고 쓰기 제한을 적용했다.
- Pillow 최소 버전을 올리고 CI 의존성 감사를 추가했다.

### 배포 환경에서 별도 실행이 필요한 항목

다음 항목은 코드 수정이 아니라 운영 계정과 원격 인프라 변경이므로 자동 적용하지 않았다.

1. Cloudflare D1 백업 생성
2. `migrations/0001_initial_schema.sql` 원격 적용
3. 삭제용·상태 변경용 운영자 토큰을 각각 생성하고 원문은 비밀번호 관리자에 저장
4. 토큰 검증 해시를 Cloudflare Pages 비밀 환경변수에 설정
5. 새 Functions와 `dist/`를 같은 배포 단위로 반영
6. 배포 후 보안 헤더, API 속도 제한, 방명록·방문 수 동작 확인

원격 설정값과 토큰 원문을 확인하지 않은 상태에서 완료로 간주하면 안 된다.

## UI/UX 우선순위와 근거

### P0 — 감상 흐름과 접근성을 막는 문제

1. 모바일 라이트박스 조작
   - 이유: 닫기·이전·다음 위치가 불명확하면 사용자가 감상 흐름을 벗어나기 어렵다.
   - 반영: 고정 닫기 버튼, 이전/다음 버튼, 현재 위치, 좌우 스와이프, 키보드 방향키를 추가했다.

2. 대화상자 포커스 관리
   - 이유: 오버레이가 열려도 포커스가 배경으로 빠지면 키보드와 보조기기 사용자가 현재 맥락을 잃는다.
   - 반영: 최초 포커스, 포커스 트랩, Escape 닫기, 배경 inert, 닫은 뒤 원래 버튼으로 포커스 복귀를 공통 모듈로 적용했다.

### P1 — 탐색 가능성과 상태 예측성을 높이는 문제

1. 사진 순서 안정화
   - 이유: 새로고침할 때마다 순서가 바뀌면 방금 본 사진을 다시 찾기 어렵다.
   - 반영: 브라우저 세션 동안 동일한 셔플 순서를 유지한다.

2. 필터·사진 URL 상태
   - 이유: 새로고침, 뒤로 가기, 링크 공유 시 탐색 상태가 사라지면 사용자가 동일한 장면으로 돌아올 수 없다.
   - 반영: 연도·권역·촬영자·장소 필터와 열린 사진 ID를 쿼리 문자열에 저장하고 `popstate`로 복원한다.

3. 지도 탐색
   - 이유: 핀이 겹치면 장소를 선택하기 어렵고, 지도만 제공하면 키보드·저시력 사용자에게 대체 경로가 부족하다.
   - 반영: 확대 수준 기반 클러스터, 권역 칩, 장소 목록, 접근 가능한 마커 이름을 추가했다.

4. 탭 의미 구조
   - 이유: 탭처럼 보이는 버튼이 탭 키보드 규칙과 상태를 제공하지 않으면 역할을 예측하기 어렵다.
   - 반영: 정보·방명록을 `tablist`/`tab`/`tabpanel` 구조로 바꾸고 지도는 별도 동작으로 분리했다.

### P2 — 피로도와 운영 실수를 줄이는 문제

1. 인트로·오디오 명시성
   - 이유: 입장 행동과 재생 행동이 모호하거나 자동 재생을 시도하면 통제감이 낮아진다.
   - 반영: 인트로 문구를 `전시 입장`으로 명확히 하고 오디오는 전용 버튼을 사용하기 전에는 재생하지 않는다.

2. 라이브 영역과 건너뛰기
   - 이유: 사진 그리드 전체가 반복 공지되면 보조기기 사용자의 탐색 비용이 커진다.
   - 반영: 그리드의 라이브 영역을 제거하고 간단한 결과 상태만 공지하며 갤러리 건너뛰기 링크를 추가했다.

3. 관리자 대량 편집
   - 이유: 모든 편집 폼이 한 번에 펼쳐지면 사진 수가 늘수록 검색과 저장 여부 확인이 어렵다.
   - 반영: 검색, 접힌 편집기, 미저장 표시·이탈 경고, 게시 체크리스트, 고급 위치 정보 접기를 추가했다.

## 결론

코드 수준의 보안 조치와 우선순위 UI/UX 개선은 반영했다. 실제 공개 환경의 보안 완료 여부는 원격 D1 마이그레이션, 비밀 환경변수 설정, Functions 재배포와 배포 후 점검까지 끝난 시점에 확정할 수 있다.
